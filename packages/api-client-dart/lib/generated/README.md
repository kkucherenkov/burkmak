# app_api_client (EXPERIMENTAL)
Spec-first HTTP API. Paths include the versioned prefix (`/api/v1`) so
generated clients can be used without additional base-URL rewriting.

**Authentication endpoints** live under `/api/v1/auth/_*` and follow the
[Better Auth protocol](https://www.better-auth.com/docs). They are
deliberately not enumerated here: Better Auth owns that wire contract and
evolves it independently of our OpenAPI surface. The `cookieAuth` and
`bearerAuth` security schemes documented below describe how the *rest* of
this API accepts credentials issued by that namespace — session cookies
(web) and bearer tokens (mobile, via the `bearer` plugin).

**Kobo native-sync endpoints** live under `/api/v1/kobo/{token}/_*` and
emulate the proprietary Kobo store API (`/v1/initialization`,
`/v1/library/sync`, reading state, covers, KEPUB download) so a stock Kobo
pointed at that mount via `api_endpoint` in `Kobo eReader.conf`
auto-syncs extracted articles. Like the auth namespace, they are not
enumerated here: the wire contract is Kobo's (device-defined, JSON,
case-sensitive), authenticated by a personal access token embedded in the
path because the device cannot send auth headers. No generated client
consumes them.
