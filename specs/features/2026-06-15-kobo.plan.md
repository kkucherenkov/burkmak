# Kobo Sync (OPDS + EPUB/KEPUB) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate a valid EPUB3/KEPUB from an extracted article and serve an OPDS 1.2 catalog so a Kobo (or any OPDS reader) browses + downloads saved articles over wifi.

**Architecture:** A `kobo` CQRS-light module: pure builders (`epub.builder.ts` via `jszip`, `kepub.transform.ts` span injection, `opds.feed.ts` Atom XML) + a `KoboController` with `GET items/:id/epub` (binary stream, disk-cached like the image cache) and `GET opds` (atom-xml). Auth reuses the `SessionGuard` (which now accepts PAT via Bearer/Basic from the `pat` slice).

**Tech Stack:** NestJS 11, Prisma 7, `jszip` (new dep), Node `fs` streams. No new DB model.

**Spec:** [2026-06-15-kobo-sync.md](./2026-06-15-kobo-sync.md). Contract (`getItemEpub`, `getOpdsFeed`, `patBasicAuth`) already in `openapi.yaml` + clients (0.4.0).

**Branch:** `feat/kobo` off `main` (after `pat` merges). Commit per task; `--no-ff` when gates pass.

**Depends on:** `pat` slice (guard PAT acceptance). Reads the S2 `Article` + on-disk image cache.

## Task 2: EPUB builder (pure, TDD)

**Files:** `epub.builder.ts`, `test/kobo.spec.ts`.

`buildEpub(input)` where `input = { item: { id, title, url, siteName, lang? }, article: { contentHtml }, images: { name: string; mediaType: string; data: Uint8Array }[] }` → `Promise<Uint8Array>` (a zip). Image `src` in `contentHtml` of the form `/api/v1/items/<id>/image/<key>` is rewritten to `images/<name>`.

- [ ] **Step 1: Failing test**

```ts
// apps/backend/test/kobo.spec.ts
import JSZip from 'jszip';
import { describe, expect, it } from 'vitest';
import { buildEpub } from '../src/modules/kobo/infra/epub.builder';

const base = {
  item: { id: 'it1', title: 'Reading Slowly', url: 'https://x.com/a', siteName: 'x.com' },
  article: { contentHtml: '<p>Hello &amp; welcome</p>' },
  images: [] as { name: string; mediaType: string; data: Uint8Array }[],
};

describe('buildEpub', () => {
  it('produces a valid epub zip with mimetype stored first', async () => {
    const bytes = await buildEpub(base);
    const zip = await JSZip.loadAsync(bytes);
    expect(await zip.file('mimetype')!.async('string')).toBe('application/epub+zip');
    expect(zip.file('META-INF/container.xml')).toBeTruthy();
    expect(zip.file('OEBPS/content.opf')).toBeTruthy();
    expect(zip.file('OEBPS/nav.xhtml')).toBeTruthy();
    const ch = await zip.file('OEBPS/chapter.xhtml')!.async('string');
    expect(ch).toContain('Hello'); // body carried over
    const opf = await zip.file('OEBPS/content.opf')!.async('string');
    expect(opf).toContain('urn:burkmak:it1'); // identifier
    expect(opf).toContain('Reading Slowly');
  });

  it('embeds images and rewrites their src', async () => {
    const bytes = await buildEpub({
      ...base,
      article: { contentHtml: '<p><img src="/api/v1/items/it1/image/abc.png"></p>' },
      images: [{ name: 'abc.png', mediaType: 'image/png', data: new Uint8Array([1, 2, 3]) }],
    });
    const zip = await JSZip.loadAsync(bytes);
    expect(zip.file('OEBPS/images/abc.png')).toBeTruthy();
    const ch = await zip.file('OEBPS/chapter.xhtml')!.async('string');
    expect(ch).toContain('images/abc.png');
    expect(ch).not.toContain('/api/v1/items/');
  });
});
```

- [ ] **Step 2: Run → FAIL.** `pnpm --filter @app/backend exec vitest run test/kobo.spec.ts`

- [ ] **Step 3: Implement `epub.builder.ts`.** Use `jszip`. Key points:
  - First entry MUST be `mimetype`, content `application/epub+zip`, added with `{ compression: 'STORE' }`.
  - `META-INF/container.xml` → rootfile `OEBPS/content.opf`.
  - `OEBPS/content.opf` (EPUB3): `<dc:identifier id="bookid">urn:burkmak:<id></dc:identifier>`, `<dc:title>`, `<dc:language>` (input.lang ?? 'en'), `<dc:source>`=url, `<dc:creator>`=siteName, `<meta property="dcterms:modified">` (pass a fixed ISO via input or derive from article; tests don't assert it — use `new Date(0).toISOString()` is fine but prefer a passed `modified` string to stay deterministic — add `modified?: string` to input, default `'1970-01-01T00:00:00Z'`). Manifest lists `chapter.xhtml`, `nav.xhtml` (with `properties="nav"`), and each image (id, href `images/<name>`, media-type). Spine references chapter.
  - `OEBPS/nav.xhtml` — minimal EPUB3 nav with one link to chapter.
  - `OEBPS/chapter.xhtml` — wrap `contentHtml` in a valid XHTML doc (`<?xml …?><!DOCTYPE html><html xmlns…><head><title>…</title></head><body>…</body></html>`); rewrite `src="/api/v1/items/<id>/image/<key>"` → `src="images/<key>"` (regex on the known prefix). XML-escape the `<title>`.
  - Add image bytes at `OEBPS/images/<name>` (default DEFLATE).
  - Return `zip.generateAsync({ type: 'uint8array' })`.

- [ ] **Step 4: Run → PASS.**

- [ ] **Step 5: Commit** — `git commit -m "feat(backend): epub3 builder (kobo)"`

## Task 3: KEPUB span injection (pure, TDD)

**Files:** `kepub.transform.ts`, append `test/kobo.spec.ts`.

`toKepubXhtml(chapterXhtml: string): string` wraps text runs in `<span class="koboSpan" id="kobo.<p>.<s>">…</span>` for Kobo progress. Keep it conservative: wrap text inside block elements (`p`, `li`, `h1..h6`), incrementing paragraph + segment counters; never wrap inside existing tags' attributes; don't double-wrap already-`koboSpan`-wrapped text.

- [ ] **Step 1: Failing test**

```ts
import { toKepubXhtml } from '../src/modules/kobo/infra/kepub.transform';
it('wraps paragraph text in koboSpans, preserving text', () => {
  const out = toKepubXhtml('<body><p>Hello world</p></body>');
  expect(out).toContain('class="koboSpan"');
  expect(out).toMatch(/id="kobo\.\d+\.\d+"/);
  expect(out).toContain('Hello world');
});
it('does not double-wrap', () => {
  const once = toKepubXhtml('<body><p>Hi</p></body>');
  expect(toKepubXhtml(once)).toBe(once);
});
```

- [ ] **Step 2–4:** Run FAIL → implement (a small tokenizer/regex over block text nodes; if `koboSpan` already present, return input unchanged) → run PASS.

## Task 6: KoboController + module + endpoints

**Files:** `kobo.controller.ts`, `kobo.module.ts`, `app.module.ts`.

- [ ] **Controller** (`@Controller({ version: '1' })`, `@UseGuards(SessionGuard)`):
  - `@Get('items/:id/epub')` `getEpub(@Req() req, @Param('id') id, @Query('format') format, @Res() res)`: `variant = format === 'epub' ? 'epub' : 'kepub'` (default kepub). Call `build-epub.service`; `not_found` → 404, `not_ready` → 409. On success set `Content-Type: application/epub+zip`, `X-Content-Type-Options: nosniff`, `Content-Disposition: attachment; filename="<slug>.kepub.epub"`, `createReadStream(path).pipe(res)` (reuse the `getImage` streaming pattern). Slug from item title.
  - `@Get('opds')` `getOpds(@Req() req, @Res() res)`: list the user's extracted items (items repo filtered to `extractStatus==='ready'`, exclude archived, newest first), `buildOpdsFeed(...)` with `baseUrl` from `AppConfig` (public API base), set `Content-Type: application/atom+xml;profile=opds-catalog;kind=acquisition` and send.
