# S2 Web — Reader View & Highlight Authoring Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** The web reading experience — `/items/[id]` becomes a full reader: trigger extraction (live `extracting → ready` via SSE), read the cleaned article, select text to create color highlights with optional notes, and manage them in a side panel. FTS body search comes for free (backend change; the existing library `q` now matches article bodies).

**Architecture:** Same Nuxt 4 SPA patterns as shipped S1-web (read `apps/web/app/{composables,pages/items/[id].vue}`): typed `useApi` over `@app/specs`, `useState` composables, `useEvents` SSE, `@app/ui` composites, `.ts` langDir i18n (NOT `<i18n>` SFC blocks — those break the build; add keys to `apps/web/i18n/locales/{en,ru}.ts`). The highlight anchor (quote+prefix+suffix) is computed by a **pure, tested util** from the browser Selection; re-anchoring/marking is `AppArticleReader`'s job (S2-ui).

**Tech Stack:** Nuxt 4 (`ssr:false`), `@app/ui` (S2 reader/highlight composites), `@nuxtjs/i18n` (en/ru, `.ts` locales), Vitest (happy-dom).

**Depends on:** **S2-ui** (`AppArticleReader`, `AppExtractState`, `AppHighlightPopover`, `AppHighlightCard`) + **S2-backend** (extract/article/highlights API + FTS, regenerated `@app/specs`). Read the shipped `apps/web/app/pages/items/[id].vue` (S1 metadata detail — this replaces its body) and `apps/web/app/composables/useItems.ts`/`useEvents.ts`.

## Task 2: Pure highlight-anchor util (TDD)
**Files:** Create `apps/web/app/utils/highlight-anchor.ts`, `apps/web/tests/unit/highlight-anchor.spec.ts`.
- [ ] **Step 1: Failing spec** — `anchorFromSelection(text, start, end, ctx=32)` returns `{ quote, prefix, suffix }` (pure, string-based so it's testable without a real Selection):
```ts
import { describe, it, expect } from 'vitest';
import { anchorFromSelection } from '../../app/utils/highlight-anchor';

it('extracts quote + bounded prefix/suffix', () => {
  const text = 'Alpha beta gamma delta epsilon.';
  const a = anchorFromSelection(text, 6, 16); // "beta gamma"
  expect(a.quote).toBe('beta gamma');
  expect(a.prefix.endsWith('Alpha ')).toBe(true);
  expect(a.suffix.startsWith(' delta')).toBe(true);
});
it('clamps context at the bounds', () => {
  const a = anchorFromSelection('Hi there world', 0, 2);
  expect(a.quote).toBe('Hi');
  expect(a.prefix).toBe('');
});
```
- [ ] **Step 2: Run → FAIL. Implement** — slice `quote = text.slice(start,end)`, `prefix = text.slice(max(0,start-ctx), start)`, `suffix = text.slice(end, end+ctx)`. (The page wires the real DOM `Selection` → text offsets, then calls this; the offset extraction is thin glue in the page, the bounded-context logic is here + tested.)
- [ ] **Step 3: Run → PASS, commit.** `git commit -m "feat(web): pure highlight-anchor util"`. Tick active.md.
