# S2 Backend — Extraction, Article, FTS5, Highlights Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Backend for S2 — on-demand `extract_article` (Readability → sanitized HTML + text + word/reading-time, lead+inline images cached locally) with live SSE status; the `Article` read model; FTS5 full-text search over title+url+body; per-user highlights CRUD. All ownership-scoped.

**Architecture:** Spec-first (edit `openapi.yaml` → codegen). Hexagonal CQRS exactly like the shipped `modules/items` (controllers dispatch Commands/Queries; handlers inject repo **ports** implemented by Prisma adapters in `infra/`). Extraction + image caching sit behind ports (`IArticleExtractor`, `IImageCache`) so impls can be upgraded. The `extract_article` handler implements S0's `JobHandler` and registers with `JobWorker`, mirroring the shipped `fetch_metadata` handler. FTS5 is a raw-SQL virtual table + triggers (Prisma can't model it), queried via `$queryRaw`.

**Tech Stack:** NestJS 11 + CQRS, Prisma 7 (SQLite/better-sqlite3, FTS5), `@mozilla/readability` + `linkedom` + `sanitize-html` (extraction), Better Auth `SessionGuard`, RxJS SSE, Vitest.

**Depends on:** S1 backend (shipped — `Item`, `ItemRepo`, `JobsService`, `JobWorker.register`, `EventsService`, `SessionGuard`, the `fetch_metadata` handler to mirror, the `HttpMetadataFetcher` to reuse). Read these before starting: `apps/backend/src/modules/items/**` and the `specs/features/2026-06-13-s1-backend.plan.md` (the pattern this mirrors). **Out of scope:** web (S2-web), mobile (S2-mobile), reader UI.

### Task 2: FTS5 virtual table + triggers + bootstrap
**Files:** Create `apps/backend/prisma/sql/item_fts.sql`, `apps/backend/src/common/fts/fts.bootstrap.ts`; wire into app startup.
- [ ] **Step 1: Write the FTS SQL** (`item_fts.sql`) — contentless-ish standalone table keyed by `item_id`, kept in sync by triggers:
```sql
CREATE VIRTUAL TABLE IF NOT EXISTS item_fts USING fts5(
  title, url, body, item_id UNINDEXED, tokenize = 'porter unicode61'
);
-- seed/refresh helpers run from triggers:
CREATE TRIGGER IF NOT EXISTS item_fts_ai AFTER INSERT ON item BEGIN
  INSERT INTO item_fts(title, url, body, item_id)
  VALUES (new.title, new.url, '', new.id);
END;
CREATE TRIGGER IF NOT EXISTS item_fts_au AFTER UPDATE ON item BEGIN
  UPDATE item_fts SET title = new.title, url = new.url WHERE item_id = new.id;
END;
CREATE TRIGGER IF NOT EXISTS item_fts_ad AFTER DELETE ON item BEGIN
  DELETE FROM item_fts WHERE item_id = old.id;
END;
CREATE TRIGGER IF NOT EXISTS article_fts_ai AFTER INSERT ON article BEGIN
  UPDATE item_fts SET body = new.contentText WHERE item_id = new.itemId;
END;
CREATE TRIGGER IF NOT EXISTS article_fts_au AFTER UPDATE ON article BEGIN
  UPDATE item_fts SET body = new.contentText WHERE item_id = new.itemId;
END;
```
- [ ] **Step 2: Bootstrap applier** (`fts.bootstrap.ts`) — split the SQL on `;` and `prisma.$executeRawUnsafe` each statement; idempotent (`IF NOT EXISTS`). Call it `OnModuleInit` from a small `FtsModule` (global) OR from `PrismaService.onModuleInit` after connect. Also backfill existing rows once: `INSERT INTO item_fts(title,url,body,item_id) SELECT i.title,i.url,COALESCE(a.contentText,''),i.id FROM item i LEFT JOIN article a ON a.itemId=i.id WHERE i.id NOT IN (SELECT item_id FROM item_fts)`.
- [ ] **Step 3: Verify FTS5 present.** At bootstrap, `prisma.$queryRawUnsafe("SELECT fts5(?)", '...')` is fragile — instead wrap the `CREATE VIRTUAL TABLE` in try/catch and log a clear error if FTS5 is unavailable. (better-sqlite3 ships FTS5; this is a guard.)

- [ ] **Step 4: Implement `article.ts`** (pure; linkedom DOM + Readability + sanitize-html):
```ts
import { Readability } from '@mozilla/readability';
import { parseHTML } from 'linkedom';
import sanitizeHtml from 'sanitize-html';

export interface ParsedArticle {
  title: string | null;
  contentHtml: string;
  contentText: string;
  wordCount: number;
  readingTimeMin: number;
}

const ALLOWED_TAGS = sanitizeHtml.defaults.allowedTags.concat(['img', 'figure', 'figcaption', 'h1', 'h2']);

export function parseArticle(html: string, url: string, wpm = 200): ParsedArticle {
  const { document } = parseHTML(html);
  // Readability needs a document with a base URL for relative links/images.
  const reader = new Readability(document as unknown as Document);
  const result = reader.parse(); // { title, content, textContent, ... } | null
  const rawHtml = result?.content ?? '';
  const contentHtml = sanitizeHtml(rawHtml, {
    allowedTags: ALLOWED_TAGS,
    allowedAttributes: { a: ['href'], img: ['src', 'alt'] },
    transformTags: { a: sanitizeHtml.simpleTransform('a', { rel: 'noopener noreferrer', target: '_blank' }) },
  });
  const contentText = (result?.textContent ?? '').replace(/\s+/g, ' ').trim();
  const wordCount = contentText ? contentText.split(' ').length : 0;
  const readingTimeMin = Math.max(1, Math.ceil(wordCount / wpm));
  return { title: result?.title ?? null, contentHtml, contentText, wordCount: contentText ? wordCount : 0, readingTimeMin: contentText ? readingTimeMin : 1 };
}
```
> Verify the Readability + linkedom `document` typing — linkedom's document is structurally compatible; cast as shown if TS complains. Resolve relative image src against `url` in `parseArticle` (set a `<base href>` on the linkedom doc before Readability, or rewrite in Task 5's image cache). Keep it pure (no IO).
