# S1 Core Library — `@app/ui` Composite Components Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the six composite components the S1 web library needs to `@app/ui` — `AppStatusBadge`, `AppTagChip`, `AppSkeleton`, `AppEmptyState`, `AppItemCard`, `AppFilterBar` — each Storybook-first with a colocated Vitest spec, built only from design tokens, composing the existing primitives.

**Architecture:** Each component is a native Vue 3 `<script setup>` SFC with scoped SCSS that references only `var(--token)` custom properties. Leaf composites (`AppStatusBadge`, `AppTagChip`) thin-wrap an existing primitive (`AppBadge`, `AppChip`); `AppItemCard`/`AppFilterBar` compose several primitives. **Translated strings are never produced inside `@app/ui`** — every user-facing string arrives as a prop, so the web app owns i18n. Visual markup is translated from the in-repo reference mockups under `specs/design/mockups/` (cited by line range per task); the plan gives the full `<script>` contract + test code and points at the mockup for the `<template>`/`<style>` detail.

**Tech Stack:** Vue 3.5 SFC, TypeScript, scoped SCSS on DTCG tokens, Vitest + `@vue/test-utils` (`happy-dom`, `globals:true`), Storybook 8 CSF3 with blocking axe a11y (`parameters.a11y.test = 'error'`).

**Depends on:** nothing in S1 code (primitives already exist). **Consumed by:** S1-web (the library/detail pages import these). **Out of scope:** auth-only widgets (`AppBrand`/`AppFormError`/`AppStrength` — built inline in S1-auth web pages), the item-detail reader/`AppReaderPreview` + fetch state machine (S2), `AppAddBar`/`AppModal` (built in the S1-web library page, not as `@app/ui` exports).

## Conventions (read once — every task obeys these)

From the existing primitives (`packages/ui/src/components/AppChip`, `AppBadge`, `AppCard`):

- **4 colocated files per component:** `packages/ui/src/components/<Name>/{<Name>.vue, <Name>.spec.ts, <Name>.stories.ts, index.ts}`.
- **Script:** `<script setup lang="ts">` (2-space-indented body). Props via `withDefaults(defineProps<{…}>(), {…})` with an explicit default for every optional prop. Emits via `const emit = defineEmits<{ name: [payload: T] }>()` (tuple-payload syntax). Class binding via a `computed` returning an **array**: `['app-foo', \`app-foo--\${variant}\`, { 'app-foo--on': on }]`.
- **Template:** BEM only — root `.app-<name>`, elements `&__el`, modifiers `&--mod`, nesting ≤3. **No Tailwind utility classes, no `style=""` with literal values, no `!important`** in component DOM. `data-testid` must pass through (don't set `inheritAttrs:false` unless attrs must land on an inner node).
- **Style:** `<style lang="scss" scoped>` (2-space body). Every value is `var(--token)`. Component-local non-token magic numbers are allowed only as SCSS `$vars` carrying a `// WHY` comment.
- **Tokens — authoritative names** (these are the `var(--…)` names committed components actually consume; copy them, do NOT invent from the token JSON which uses different role keys):
  - Brand: `--brand-accent`, `--brand-accent-hover`, `--brand-accent-active`, `--brand-accent-soft`, `--brand-accent-soft-hover`, `--brand-accent-fg`, `--brand-accent-border`.
  - Surface: `--surface-bg`, `--surface-bg-subtle`, `--surface-bg-muted`, `--surface-surface`, `--surface-surface-alt`, `--surface-overlay`.
  - Text: `--text-fg`, `--text-fg-muted`, `--text-fg-subtle`, `--text-fg-disabled`, `--text-fg-inverse`.
  - Border: `--border-default`, `--border-strong`, `--border-soft`.
  - Status (each + `-soft`): `--status-success`, `--status-warning`, `--status-danger`, `--status-info` (note: danger, not error).
  - Space `--space-1…--space-24` (4px grid); Radius `--radius-{xs,sm,md,lg,xl,2xl,pill}`; Shadow `--shadow-{xs,sm,md,lg,focus}` (use `--shadow-focus` for `:focus-visible`).
  - Type: families `--font-display` (Fraunces), `--font-reading` (Literata), `--font-sans` (Hanken, default), `--font-mono`; sizes `--text-{xs,sm,md,base,lg,xl,2xl…}`; weights `--fw-{regular,medium,semibold,bold}`; leading `--leading-{tight,snug,normal,relaxed}`; tracking `--tracking-{tight,snug,normal,wide}`.
  - Motion: `--dur-{fast,base,slow}`, `--ease`/`--ease-default`/`--ease-out`; `--lift-{sm,md}`. Opacity: `--opacity-disabled`.
- **Stories:** CSF3, `title: 'Compositions/<Name>'`, `tags: ['autodocs']`. Light/dark/sepia/forest is a **global toolbar toggle** (`.storybook/preview.ts`) — do NOT add per-story theme variants. a11y is **blocking** — every story must pass axe. Matrix stories may use inline `style="… var(--space-4) …"` (stories only, never component DOM).
- **Primitive APIs you'll compose:**
  - `AppBadge` — props `label?`, `color?: primary|neutral|success|warning|error|info` (def neutral), `variant?: solid|outline|soft|subtle` (def subtle), `size?: sm|md|lg`, `icon?`. No emits.
  - `AppChip` — props `label?`, `color?` (6), `variant?` (def soft), `size?`, `icon?`, `dismissible?`, `selected?` (→`aria-pressed`), `disabled?`, `to?`. Emits `click:[MouseEvent]`, `dismiss:[Event]`.
  - `AppCard` — props `title?`, `description?`, `interactive?`. Emits `click:[Event]`. Slots `header`, default, `footer`.
  - `AppIcon` — props `name:string` (iconify, e.g. `i-lucide-star`), `size?: xs|sm|md|lg|xl`, `label?` (omit → decorative `aria-hidden`).
  - `AppButton` — props `label?`, `variant?: solid|outline|ghost|link`, `color?: primary|neutral|success|warning|error`, `size?`, `loading?`, `disabled?`, `icon?`, `iconTrailing?`. Emits `click:[MouseEvent]`.
  - `AppInput` — props `modelValue:string` (req), `type?`, `placeholder?`, `size?`, `disabled?`. Emits `update:modelValue:[string]`.
  - `AppSelect` — props `modelValue:string|null` (req), `options:readonly AppSelectOption[]` (req; `{id,label,disabled?}`), `placeholder?`, `size?`. Emits `update:modelValue:[string]`.

<script setup lang="ts">
  import { computed } from 'vue';

  import AppBadge from '../AppBadge/AppBadge.vue';

  type Status = 'pending' | 'ready' | 'failed';

  const props = defineProps<{ status: Status; label: string }>();

  const color = computed(() =>
    props.status === 'ready' ? 'success' : props.status === 'failed' ? 'error' : 'info',
  );
  const icon = computed(() =>
    props.status === 'ready'
      ? 'i-lucide-check'
      : props.status === 'failed'
        ? 'i-lucide-triangle-alert'
        : 'i-lucide-loader',
  );
  const rootClasses = computed(() => ['app-status-badge', `app-status-badge--${props.status}`]);
</script>

<template>
  <AppBadge
    :class="rootClasses"
    :color="color"
    variant="soft"
    size="sm"
    :icon="icon"
    :label="label"
  />
</template>

<style lang="scss" scoped>
  // Pulsing affordance on pending — translate the `app-pulse` dot from

  .app-status-badge {

  }

  @keyframes app-status-pulse {

      opacity: var(--opacity-disabled);
    }

<script setup lang="ts">
  import { computed } from 'vue';

  import AppChip from '../AppChip/AppChip.vue';
  import AppInput from '../AppInput/AppInput.vue';
  import AppSelect, { type AppSelectOption } from '../AppSelect/AppSelect.vue';

  export type AppFilterSegment = 'unread' | 'read' | 'archived' | 'favorite';

  const props = defineProps<{
    segment: AppFilterSegment;
    q: string;
    tag: string | null;
    tagOptions: readonly AppSelectOption[];
    labels: {
      unread: string;
      read: string;
      archived: string;
      favorite: string;
      allTags: string;
      search: string;
    };
  }>();

  const emit = defineEmits<{
    'update:segment': [value: AppFilterSegment];
    'update:q': [value: string];
    'update:tag': [value: string | null];
  }>();

  const segments = computed<{ key: AppFilterSegment; label: string }[]>(() => [
    { key: 'unread', label: props.labels.unread },
    { key: 'read', label: props.labels.read },
    { key: 'archived', label: props.labels.archived },
    { key: 'favorite', label: props.labels.favorite },
  ]);
</script>

<template>
  <div class="app-filter-bar">
    <div class="app-filter-bar__segments" role="group">
      <AppChip
        v-for="s in segments"
        :key="s.key"
        class="app-filter-bar__segment"
        :label="s.label"
        :selected="segment === s.key"
        color="primary"
        variant="soft"
        size="sm"
        @click="emit('update:segment', s.key)"
      />
    </div>

    <AppSelect
      class="app-filter-bar__tags"
      :model-value="tag"
      :options="tagOptions"

      :model-value="q"
      :placeholder="labels.search"
      size="sm"
      @update:model-value="emit('update:q', $event)"
    />
  </div>
</template>
