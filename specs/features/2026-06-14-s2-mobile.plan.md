# S2 Mobile — Reader & Read-Only Highlights Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** The mobile reading experience — the item-detail screen gains Extract (live `extracting → ready` via SSE) and renders the cleaned article with `flutter_html`, showing the user's highlights **read-only** as colored marks. Search over the article body comes free (backend FTS via the existing `q`).

**Architecture:** Same feature-first Flutter patterns as shipped S1-mobile (read `apps/mobile/lib/features/item_detail/**` and `library/**`): `flutter_bloc` cubits, `get_it` DI, the generated `app_api_client`, slang i18n. Article HTML is rendered with `flutter_html`. Highlights are applied read-only by **injecting `<mark>` tags** into `contentHtml` (quote+prefix+suffix search — a pure, tested util) and styling them in `flutter_html`. Live extract status uses the S1 `EventsClient` SSE (the cubit subscribes for this item), exactly like `LibraryCubit`.

**Tech Stack:** Flutter 3.41, `flutter_bloc`/`equatable`, `get_it`, `flutter_html`, generated `app_api_client` (extract/article/highlights endpoints), slang (en/ru/uk/el); tests `bloc_test`. Flutter SDK: `~/projects/petProjects/tools/flutter/bin`.

**Depends on:** **S2-backend** (extract/article/highlights API + regenerated `app_api_client`). Read the shipped `apps/mobile/lib/features/item_detail/presentation/{item_detail_screen.dart,bloc/*}`, `library/data/items_repository.dart`, `shared/network/events_client.dart`. No highlight authoring (S2 cut — web only).

```dart
test('wraps the quoted text in a colored mark', () {
  final out = injectHighlights('<p>Alpha beta gamma.</p>',
      [const HlAnchor(quote: 'beta', prefix: 'Alpha ', suffix: ' gamma', color: 'yellow')]);
  expect(out, contains('<mark class="hl-yellow">beta</mark>'));

test('leaves html unchanged when the quote is absent', () {
