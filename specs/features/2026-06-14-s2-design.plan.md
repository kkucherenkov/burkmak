# S2 Design — Highlight & Reader Mockups Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Produce the missing S2 design mockups — the highlight-create popover, the highlights/notes panel, and the note editor — so the S2-ui components have a visual source of truth (the reader/fetch machine is already designed in `item-detail.vue`).

**Architecture:** Mockups are standalone reference `.vue` files under `specs/design/mockups/` (the same form as the S1 mockups), using the emitted design tokens (`var(--…)` names from `packages/ui/src/tokens.generated.css`) — NOT shipped components. They are the visual contract the `@app/ui` S2 components are translated from.

**Tech Stack:** Standalone Vue SFC mockups, scoped SCSS on design tokens (post-reconciliation emitted names), the burkmak brand (Fraunces/Literata/Hanken, clay accent) per `specs/design/DESIGN.md`.

**Depends on:** the design tokens + DESIGN.md (shipped). **Consumed by:** S2-ui.

---

## Conventions
- Mockups go in `specs/design/mockups/` next to the S1 set (`item-detail.vue`, `library.vue`, …). Read `specs/design/mockups/item-detail.vue` first — match its file shape (a self-contained SFC with a small script driving demo state + commented `@app/ui:` annotations marking which composite each block maps to).
- **Tokens only** — use the emitted names (`--brand-accent[-subtle]`, `--surface-page/-surface/-raised/-overlay`, `--text-fg/-secondary/-tertiary`, `--border-default/-strong`, `--status-{success,warning,error,info}-{fg,subtle}`, `--font-display/-reading/-sans`, `--fw-*`, `--text-*`, `--space-1..32`, `--radius-*`, `--shadow-{sm,md,lg}`, `--dur-*`, `--ease`). No raw hex, no `--*-soft`/`--surface-bg*` (those don't exist).
- Highlight colors: `yellow|green|blue|pink` — pick token-friendly swatch values (use `--status-*` family or DESIGN.md accents; document the chosen hex→role in a comment since highlight swatches are content colors, not chrome).
- These are mockups, not `@app/ui` — no Storybook/spec required.

## File Structure
```
specs/design/mockups/
```

(A single mockup file is fine — it composes all three new pieces in one reader scene, matching how `item-detail.vue` composes the reader. Split into 2 files only if it grows unwieldy.)

## Task 1: Reader-with-highlights mockup

**Files:** Create `specs/design/mockups/reader-highlights.vue`.

- [ ] **Step 1: Read the references.** Read `specs/design/mockups/item-detail.vue` (reader column + fetch machine + the `AppReaderToolbar`/`app-reader-preview` markup) and `specs/design/DESIGN.md` (reading-room aesthetic, Literata reader body, clay accent). Note the `.app-reader*` and `.app-fetch*` class conventions already used.

- [ ] **Step 2: Build the reader scene.** Compose, in one SFC, the post-extraction reading experience:
  1. **Reader column** — a `ready`-state article (Literata body, generous leading, `~44rem` measure) with 2–3 paragraphs of lorem, including **2 highlight marks** (`.app-highlight--yellow`, `.app-highlight--green`) inline so the mark styling is visible. Annotate `<!-- @app/ui: AppArticleReader -->`.
  2. **Selection popover** — a small floating card (`.app-highlight-popover`) anchored above a selected phrase: 4 color swatches (yellow/green/blue/pink) + an "Add note" affordance. Annotate `<!-- @app/ui: AppHighlightPopover -->`. Show it positioned over one selected run of text.
  3. **Highlights side panel** — a column/drawer (`.app-highlights-panel`) listing 2–3 `.app-highlight-card`s: each = colored quote + optional note + edit/delete affordances + timestamp. Annotate `<!-- @app/ui: AppHighlightCard -->`.
  4. **Note editor** — an inline expanded state of one `app-highlight-card` showing a textarea for the note + save/cancel. Annotate `<!-- @app/ui: AppHighlightCard (editing) -->`.
  A tiny `<script setup>` may hold demo state (which highlight is selected/editing) to make the scene render coherently — mockup-only, no real logic.

- [ ] **Step 3: Style it (tokens only).** Reader body `--font-reading`/`--leading-relaxed`/`--text-fg`; highlight marks use soft content-color backgrounds (document the 4 swatch hexes in a comment, mapped to yellow/green/blue/pink); popover/panel/cards use `--surface-raised`, `--border-default`, `--radius-lg`, `--shadow-md`; actions use `--brand-accent`. Responsive: panel collapses below the reader on narrow widths. Match `item-detail.vue`'s quality bar.

- [ ] **Step 4: Validate + commit.** Mockups aren't linted as app code, but run `pnpm format` so it's prettier-clean. Confirm it's self-contained (no imports of real components — it stands alone like the other mockups).

```bash
pnpm format
git add specs/design/mockups/reader-highlights.vue
git commit -m "design(s2): reader + highlights/notes mockup"
```
