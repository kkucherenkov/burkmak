# Personal Access Tokens — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A long-lived `burk_pat_…` token a user generates in Settings and pastes into headless clients (Kobo OPDS, Obsidian plugin); the auth guard accepts it via `Authorization: Bearer` and HTTP Basic, resolving `req.userId` like a session.

**Architecture:** New `PersonalAccessToken` Prisma model (stores only a SHA-256 hash). A `tokens` CQRS module (create/list/revoke) following the existing items/highlights module shape. A `PatService.resolve(req)` the existing `SessionGuard` calls after the Better-Auth session check. Web Settings gets a tokens card using the regenerated `@app/api-client-ts` operations (`createToken`/`listTokens`/`revokeToken`).

**Tech Stack:** NestJS 11 + CQRS, Prisma 7 (SQLite), Better Auth, Node `crypto`; Nuxt 4 web + `@app/ui`; Vitest.

**Spec:** [2026-06-15-personal-access-tokens.md](./2026-06-15-personal-access-tokens.md). **Contract** (tokens paths/schemas) is already in `openapi.yaml` + regenerated clients.

**Branch:** `feat/pat` off `main`. This slice lands BEFORE `kobo`/`obsidian` (they consume the guard). Commit per task; `--no-ff` merge when gates pass.

```ts
// apps/backend/src/modules/tokens/infra/pat.crypto.ts
import { createHash, randomBytes } from 'node:crypto';

export const PAT_PREFIX = 'burk_pat_';

export function hashToken(secret: string): string {
  return createHash('sha256').update(secret).digest('hex');
}

export function generateToken(): { secret: string; hash: string; prefix: string } {
  const secret = PAT_PREFIX + randomBytes(32).toString('base64url');
  return { secret, hash: hashToken(secret), prefix: secret.slice(0, PAT_PREFIX.length + 4) };
}
```

## Task 5: Guard accepts PAT (Bearer + Basic)

**Files:** `pat.service.ts` (new), `auth.guard.ts` (mod), `app.module.ts` (provide PatService).

- [ ] **Step 1: PatService test** (append to `test/tokens.spec.ts`): with a created token,
  - `resolve` of a req with header `authorization: 'Bearer ' + secret` → that userId, and `touch` called.
  - `authorization: 'Basic ' + base64('x:' + secret)` (username ignored) → userId.
  - a Better-Auth-style bearer (no `burk_pat_` prefix) → `null` (not mistaken for a PAT).
  - revoked/unknown secret → `null`.

Stub the request as `{ headers: { authorization } }`.

- [ ] **Step 2: Implement `pat.service.ts`**

```ts
import { Inject, Injectable } from '@nestjs/common';
import { PAT_PREFIX, hashToken } from '../../modules/tokens/infra/pat.crypto';
import { PAT_REPO, type PatRepo } from '../../modules/tokens/domain/tokens.ports';

@Injectable()
export class PatService {
  constructor(@Inject(PAT_REPO) private readonly repo: PatRepo) {}

  /** Extract a burk_pat_ secret from Authorization (Bearer or Basic password); null if none. */
  private extract(authHeader?: string): string | null {
    if (!authHeader) return null;
    if (authHeader.startsWith('Bearer ')) {
      const t = authHeader.slice(7).trim();
      return t.startsWith(PAT_PREFIX) ? t : null;
    }
    if (authHeader.startsWith('Basic ')) {
      const decoded = Buffer.from(authHeader.slice(6).trim(), 'base64').toString('utf8');
      const pwd = decoded.slice(decoded.indexOf(':') + 1);
      return pwd.startsWith(PAT_PREFIX) ? pwd : null;
    }
    return null;
  }

  async resolve(req: { headers: Record<string, unknown> }): Promise<string | null> {
    const header = req.headers['authorization'];
    const secret = this.extract(typeof header === 'string' ? header : undefined);
    if (!secret) return null;
    const found = await this.repo.findActiveByHash(hashToken(secret));
    if (!found) return null;
    void this.repo.touch(found.id); // best-effort, non-blocking
    return found.userId;
  }
}
```

- [ ] **Step 3: Wire into `SessionGuard`** — inject `PatService`; after the Better-Auth `getSession` check fails to produce a user, try `const userId = await this.pat.resolve(req); if (userId) { req.userId = userId; return true; }` before throwing `UnauthorizedException`. Keep session-first so cookie/bearer sessions are unchanged. Provide `PatService` (and ensure `PAT_REPO` is available to it — export from `TokensModule` or provide in the auth module).
