# S2 UI — Reader & Highlight `@app/ui` Components Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add the four S2 composite components to `@app/ui` — `AppExtractState`, `AppArticleReader`, `AppHighlightPopover`, `AppHighlightCard` — Storybook-first with colocated specs, tokens-only, strings-as-props, translated from the S2 + S1 mockups.

**Architecture:** Native Vue 3 `<script setup>` SFCs with scoped SCSS on emitted design tokens, composing existing primitives (AppButton, AppStatusBadge, AppIcon, AppTextarea, AppCard). Same conventions as the shipped S1 composites (read `packages/ui/src/components/AppItemCard/**` as the template). Strings are props; no `useI18n` inside `@app/ui`. The reader renders **server-sanitized** HTML via `v-html` (trust boundary documented; sanitization is the backend's job — S2-backend Task 4).

**Tech Stack:** Vue 3.5 SFC, scoped SCSS on DTCG tokens, Vitest + `@vue/test-utils` (happy-dom), Storybook CSF3 (blocking axe).

**Depends on:** **S2-design** (mockup `reader-highlights.vue`) + the shipped S1 `@app/ui` (read `AppItemCard`, `AppStatusBadge`, `AppButton`, `AppTextarea`, `AppChip` for the conventions). Run `pnpm design:build` first (tokens). **Consumed by:** S2-web.

## Conventions (same as S1-ui — read `specs/features/2026-06-13-s1-ui.plan.md` Conventions)
4 colocated files per component; `withDefaults(defineProps<…>())`; tuple-payload emits; BEM-only classes (no Tailwind utils / inline literal styles / `!important`); `<style lang="scss" scoped>` token-only; strings as props; CSF3 stories titled `Compositions/<Name>`, blocking axe; commands `pnpm --filter @app/ui {test,typecheck,lint,storybook}` + repo `pnpm stylelint:fix && pnpm format`. **Emitted tokens** are the only valid `var(--…)` names (see S1-ui plan list).

## Task 2: `AppArticleReader` — renders the article + highlight marks + selection
**Files:** `packages/ui/src/components/AppArticleReader/*`.
- [ ] **Step 1: Failing spec** — props `contentHtml: string` (server-sanitized), `highlights: AppHighlightData[]` (`{id, quote, prefix, suffix, color}`); emits `select: [{ quote, prefix, suffix }]` on text selection (web authoring) and `highlightClick: [id]`. Define `AppHighlightData` + `AppHighlightColor` types exported. Assert: renders the html (`w.find('.app-article-reader__body').html()` contains a known tag); applies highlight marks for each highlight whose `quote` is found in the rendered text (`.app-highlight--<color>` count === matched highlights); clicking a mark emits `highlightClick`.
```ts
// spec sketch
const html = '<p>Alpha beta gamma delta.</p>';
const highlights = [{ id: 'h1', quote: 'beta gamma', prefix: 'Alpha ', suffix: ' delta', color: 'yellow' }];
const w = mount(AppArticleReader, { props: { contentHtml: html, highlights } });
expect(w.findAll('.app-highlight--yellow')).toHaveLength(1);
await w.find('.app-highlight--yellow').trigger('click');
expect(w.emitted('highlightClick')?.[0]).toEqual(['h1']);
```
- [ ] **Step 2: Run → FAIL. Implement.** Render `contentHtml` via `v-html` into `.app-article-reader__body` (document the server-sanitized trust boundary in a `// WHY` comment + ensure no `eslint-disable` without WHY). After mount + on `highlights` change, **re-anchor** each highlight: search the body's text for `quote` (disambiguated by `prefix`/`suffix`), wrap the matched range in a `<mark class="app-highlight app-highlight--<color>" data-hl="<id>">` (use a small DOM range-wrap util; keep it in the component or a colocated `highlight-mark.ts` with its own spec). Bind a `mouseup`/`selectionchange` handler that, when there's a non-empty selection inside the body, computes `{quote, prefix, suffix}` and emits `select`. Delegate clicks on `mark[data-hl]` → `highlightClick`. Body uses `--font-reading`, `--leading-relaxed`, `~44rem` measure.
- [ ] **Step 3: Run → PASS. Stories** (`Plain`, `WithHighlights`) `title:'Compositions/AppArticleReader'`. a11y: marks are not interactive-only (keyboard handled or marks are decorative + the panel is the control surface — document the choice).
- [ ] **Step 4: index.ts (export component + `AppHighlightData`/`AppHighlightColor` types), lint/format, commit.** `git commit -m "feat(ui): AppArticleReader (renders article + highlight marks)"`. Tick active.md.
